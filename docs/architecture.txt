This document describes the current architecture of Thermal Polaris, showing how the React frontend, FastAPI backend, and supporting services connect to deliver real-time thermal camera monitoring and analytics.

FRONTEND (React / Vite)
- React SPA with pages for Dashboard, Analytics, Cameras, Recordings, Schedules, Settings, Login, Signup, Forgot Password, and Reset Password
- Zustand state stores: `authStore.js`, `cameraStore.js`
- Services: `api.js`, `notifications.js`, `websocket.js`
- Hooks: `useNotifications.js`, `useRealTimeData.js`
- Components: `Navbar.jsx`, `ConnectionBar.jsx`, `VideoPlayer.jsx`, `SchedulerModal.jsx`, `NotificationCenter.jsx`, `ProfileDropdown.jsx`
- Sends HTTP requests and manages WebSocket streams

BACKEND (FastAPI)
- Entry point: `backend/app.py`
- Uses `database.py` for SQLite ORM models and session management
- Auth: `backend/auth.py`
- Camera session handling: `backend/camera_handler.py`
- YOLO detection: `backend/yolo_detector.py`
- Recording manager: `backend/recording_manager.py`
- Smart event recording: `backend/smart_recording_manager.py`
- Scheduler service: `backend/scheduler_service.py`
- Notifications: `backend/notification_service.py`
- Email delivery: `backend/email_service.py`
- WebSocket routing and user messaging: `backend/websocket_manager.py`

Key backend responsibilities:
- Camera connect/disconnect for USB, RTSP, IP, HTTP, and raw streams
- WebSocket live streaming at `/ws/video/{session_id}`
- Real-time dashboard updates at `/ws/updates/{user_id}`
- YOLO detection and detection persistence
- Manual, scheduled, and smart recording workflows
- Screenshot capture and media storage
- Password reset and SMTP test support

Database models:
- `User`
- `Camera`
- `Recording` (supports manual, scheduled, and smart clips)
- `Detection`
- `RecordingSchedule`
- `Notification`

Storage resources:
- `recordings/`
- `smart_recordings/`
- `screenshots/`
- `detections/`
- `thumbnails/`

API endpoint groups:
- Auth: `/api/auth/*`
- Camera: `/api/camera/*`
- Recording: `/api/recording/*`
- Smart recording: `/api/smart-recording/*`
- Detection: `/api/detection/list`
- Screenshot: `/api/screenshot/capture`
- Schedule: `/api/schedule/*`
- Notifications: `/api/notifications*`
- Dashboard stats: `/api/dashboard/stats`

Data flow summary:
1. User authenticates and selects a camera.
2. Frontend calls `/api/camera/connect` to start a camera session.
3. User opens the live viewer and connects to `/ws/video/{session_id}`.
4. Backend delivers frames, detection overlays, and event messages.
5. Smart recording and schedule services capture clips around detection events.
6. Notifications and dashboard stats update in real time.

This architecture supports responsive live monitoring, AI detection, event-driven recording, and secure user workflows in a single integrated system.
